# 구글 스프레드시트 설정 가이드

## 📊 스프레드시트 구조

### 시트 이름
`fortune_data`

### 헤더 (첫 번째 행)
| A열 | B열 | C열 | D열 |
|-----|-----|-----|-----|
| date | zodiac | category | content |

### 데이터 형식 예시

| date | zodiac | category | content |
|------|--------|----------|---------|
| 2026-01-01 | rat | overall | 오늘은 새로운 시작의 날입니다... |
| 2026-01-01 | rat | money | 예상치 못한 수입이... |
| 2026-01-01 | rat | work | 업무에서 좋은 성과가... |
| ... | ... | ... | ... |

## 📝 필드 설명

### date (A열)
- 형식: `YYYY-MM-DD` (예: 2026-01-01)
- 타입: 텍스트 또는 날짜
- 설명: 해당 운세의 날짜

### zodiac (B열)
- 가능한 값:
  - `rat` (쥐띠)
  - `ox` (소띠)
  - `tiger` (호랑이띠)
  - `rabbit` (토끼띠)
  - `dragon` (용띠)
  - `snake` (뱀띠)
  - `horse` (말띠)
  - `sheep` (양띠)
  - `monkey` (원숭이띠)
  - `rooster` (닭띠)
  - `dog` (개띠)
  - `pig` (돼지띠)
- 타입: 텍스트
- 설명: 띠 종류 (영문 소문자)

### category (C열)
- 가능한 값:
  - `overall` (종합운)
  - `money` (재물운)
  - `work` (직장/사업운)
  - `health` (가정/건강운)
  - `relationship` (이성/대인관계)
- 타입: 텍스트
- 설명: 운세 카테고리 (영문 소문자)

### content (D열)
- 타입: 텍스트 (장문)
- 설명: 실제 운세 내용
- 권장: 2-4문장

## 🔧 스프레드시트 생성 단계

### 1. 새 스프레드시트 만들기
1. Google Sheets 접속
2. 빈 스프레드시트 생성
3. 제목을 원하는 이름으로 변경 (예: "띠별 운세 데이터")

### 2. 시트 설정
1. 하단의 "Sheet1" 시트 이름을 `fortune_data`로 변경
2. A1:D1에 헤더 입력:
   - A1: date
   - B1: zodiac
   - C1: category
   - D1: content

### 3. 헤더 서식 지정 (선택사항)
1. A1:D1 선택
2. 배경색: 연한 회색
3. 텍스트: 굵게
4. 정렬: 가운데

### 4. 열 너비 조정 (선택사항)
- A열 (date): 120px
- B열 (zodiac): 100px
- C열 (category): 120px
- D열 (content): 400px

### 5. 공유 설정
1. 우측 상단 "공유" 버튼 클릭
2. "일반 액세스" 섹션에서
3. "링크가 있는 모든 사용자"로 변경
4. 권한: "뷰어" (보기 전용)
5. "완료" 클릭

⚠️ **중요**: 스프레드시트를 공개로 설정해야 웹사이트에서 데이터를 읽을 수 있습니다!

### 6. 스프레드시트 ID 복사
URL에서 ID 부분을 복사합니다:
```
https://docs.google.com/spreadsheets/d/[이_부분이_ID]/edit
```

## 🤖 자동화 스크립트 작동 방식

매일 자정에 Python 스크립트가:
1. 오늘 날짜의 기존 데이터 삭제
2. 12개 띠 × 5개 카테고리 = 60개 운세 생성
3. 새로운 데이터를 스프레드시트에 추가

## 📌 참고사항

### 수동으로 운세 추가하기
- 자동 생성 전에 수동으로 특정 날짜의 운세를 미리 작성할 수 있습니다
- 같은 날짜의 데이터가 있으면 자동 스크립트가 삭제하고 새로 생성합니다
- 특정 날짜를 보존하려면 스크립트 수정이 필요합니다

### 데이터 백업
- 파일 > 버전 기록 > 변경사항 보기
- 정기적으로 복사본 만들기 권장

### 문제 해결
- 웹사이트에 데이터가 안 보이면 공유 설정 확인
- 스크립트 오류 시 GitHub Actions 로그 확인
- 데이터 형식이 정확한지 확인 (특히 날짜 형식)
